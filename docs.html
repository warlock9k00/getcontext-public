Adding the App

Installing Get Context takes about a minute and a half.

  1. Open the Zoom Marketplace and search for Get Context, or click the install link on the Marketplace listing.
  2. Click Install. Zoom shows the OAuth consent screen with the three scopes the app needs (see Permissions below).
  3. Click Authorize. Zoom redirects you to https://get.context.select/ with a confirmation message.
  4. That's it — you're installed. The app will now automatically join any Zoom meeting you host. There is nothing else to configure.

Prerequisites: a Zoom Pro account (or higher) and a GitHub repository where you would like your meeting notes to be committed.

Permissions requested

  • meeting:read:meeting — to fetch the join URL and passcode of a meeting that has just started, so the bot can join.
  • user:read:user — to read your name and email so meetings you host are correctly attributed to your account.
  • user:read:zak — to obtain a short-lived Zoom Access Key (ZAK) per meeting. The bot uses ZAK to join the meeting via the Meeting SDK on behalf of the authorizing user, never anonymously.

Get Context also subscribes to two Zoom event notifications:

  • meeting.started — triggers the bot to join.
  • meeting.ended — triggers post-processing.

Usage

Feature 1: Auto-join and record

Use case: you host a Zoom meeting and want a Markdown record of what was discussed without doing anything during the call.

Prerequisites: Get Context installed on your Zoom account; you are the host of the meeting.

  1. Start any Zoom meeting (instant, scheduled, or from a calendar invite).
  2. Within 5–10 seconds, a participant named "Meeting Notes" joins your meeting.
  3. Zoom's standard recording indicator (red REC dot in the top bar) turns on, and Zoom plays its standard audible announcement that the meeting is being recorded. This is by Zoom's design and cannot be disabled — Get Context never hides its presence.
  4. Conduct your meeting normally. The bot does not chat, mute participants, share screen, or take any action besides recording the meeting audio.
  5. End the meeting when you are done. The bot will leave automatically.

Feature 2: Transcript + AI summary in your GitHub repo

Use case: after the meeting, you want a searchable Markdown file with the full transcript, plus a TL;DR, action items, and decisions, in a private GitHub repository you control.

Prerequisites: Feature 1 above completed; a GitHub repository configured with Get Context (see Setup below).

  1. Within ~2 minutes of the meeting ending, a new file appears at sources/YYYY-MM-DD-zoom-<meeting-id>.md in your repo, containing the timestamped transcript.
  2. ~1–3 minutes later, a follow-up commit appends a TL;DR, decisions, action items, and open questions section generated by Anthropic Claude.
  3. The Markdown file is yours: it lives in your repository, is exportable, searchable in any tool (Obsidian, grep, GitHub search), and never locked into our service.

Feature 3: Telegram notification (optional)

Use case: you want a quick mobile notification with a link to the just-committed Markdown file.

Prerequisites: a Telegram bot configured for your account (contact hello@context.select for setup help).

When the AI summary commit lands, a Telegram message is sent to your configured chat with the meeting title, duration, and a direct link to the GitHub Markdown file.

Topic blocklist

You can configure topic substrings (for example "therapy", "personal") that, when present in a meeting's title, prevent the bot from joining. This is useful for meetings you'd rather keep off the record. The blocklist is configured server-side at install time; a self-service UI is on the roadmap.

Removing the App

  1. Open Zoom Marketplace → Installed Apps.
  2. Find Get Context in the list and click Uninstall.
  3. Confirm the uninstall in the dialog Zoom shows.

What happens to your data after de-authorization

When you uninstall the app, the following data implications apply:

  • OAuth tokens (your refresh token and any cached access token) are deleted from our Cloudflare KV store within 24 hours of receiving Zoom's app_deauthorized event. After deletion, the Service can no longer make any Zoom API call attributed to your account.
  • Webhook subscriptions are automatically removed by Zoom — we will stop receiving meeting.started and meeting.ended events for your account.
  • Meeting audio recorded during prior sessions is, in the normal course of operation, already deleted from our R2 bucket within 24 hours of transcription. The deauthorization itself does not retain audio longer.
  • Transcripts and summaries already committed to your GitHub repository are not affected — they remain in your repo, which you own. If you wish to delete them, do so directly in GitHub (we have no further access).
  • If you would like a written confirmation that data deletion has completed, write to privacy@context.select and we will provide one within 7 days.

Data flow

Zoom meeting.started
  → Cloudflare Worker (HMAC verify, fetch meeting URL)
  → Spawn bot via Attendee SDK on Hetzner VPS (Germany)
  → Bot joins meeting, records audio to Cloudflare R2 (EU)
  → meeting.ended
  → Sidecar pulls recording, transcodes to opus
  → Groq Whisper Large v3 → transcript with timestamps
  → Anthropic Claude Haiku → summary, action items, decisions
  → Commit Markdown to your GitHub repo
  → Audio deleted within 24 hours
  → Telegram notification

Self-host

The full source code is open source and available on GitHub:

  • meeting-bot — Cloudflare Worker (gateway and OAuth)
  • meeting-bot-sidecar — Python processor (Hetzner)

If you'd rather run your own infrastructure rather than use the hosted Service, the repos contain Docker configurations and step-by-step deployment notes.
